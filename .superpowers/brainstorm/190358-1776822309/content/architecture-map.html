Section 2: System Architecture

Master blueprint for a single-tenant, admin-curated multimodal RAG assistant for field engineers

End-to-End System Map
1. Admin Intake

Official Vendor Documents Only

Admin registers vendor, product family, manual metadata, source URL, and version notes.

2. Ingestion Pipeline

Parse, OCR, Normalize

LlamaParse handles page-level parsing. Mistral OCR is the fallback/augmentation path for difficult tables and diagrams. The pipeline extracts page text, tables, image references, and document metadata.

3. Knowledge Assets

Chunks, Tables, Diagrams, Citations

Normalized content becomes searchable chunks with page anchors, source spans, confidence markers, and asset links for viewer rendering.

4. Storage Layer

Convex as Operational Core

Convex stores documents, ingestion jobs, searchable chunk records, chat sessions, evaluation runs, and file references. Vector search runs from Convex actions.

5. Retrieval + Answering

Conservative, Citation-First Answer Flow

User query -> embedding + retrieval -> context assembly -> answer generation constrained to evidence -> return answer, citations, and supporting assets. Low-confidence cases must refuse to speculate.

6. User Experience

Astro Shell + React Islands

Astro renders the shell. React islands power admin controls, chat interaction, viewer selection, and realtime job state where needed.

Phased Thin Slices
SP1

Platform Core

One official PDF, one admin flow, one ingestion path, one ask flow, citations, and split-screen evidence viewer.

SP2

Knowledge Quality

Hybrid retrieval, metadata enrichment, tuning, and evaluation datasets for tricky engineering questions.

SP3

Admin Operations

Retry/reprocess flows, versioning, job observability, duplicate handling, and cost controls.

SP4

User Experience Depth

Chat history, saved findings, richer table rendering, document drill-down, and mobile adaptation.

SP5

Production Readiness

Security hardening, monitoring, backup/export, deployment workflow, and operational governance.

Current Boundary

Build the whole map, ship in slices

The blueprint covers the full system. The first executable spec will target SP1 only, while preserving interfaces needed by SP2-SP5.
